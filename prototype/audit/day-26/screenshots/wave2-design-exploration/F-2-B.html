設計意図

手順 markdown の本文差分は段落単位で読まれることが多く、左右 2 column の Side-by-side が default として情報構造に最も忠実。 「文中に変更がどの語で生じたか」を 1 行で確認したいときだけ Inline に切替えればよい。 MetadataStrip を PageHeader 直下に置くことで「提案レビューを開いた瞬間、何のメタが付いているか」を 0 秒で目に入れ、scroll 動作なしで gate を満たせる (CaseReview 案 A と違い、メタが文書差分の 前 に来るのが手順承認の心理 model に合う)。

採用適性

既存 ProposalReview の border-l-2 hairline + tint /20 grammar を維持しつつ、左右 column 化で Card 2 ★ の 3-view 要件 (Side-by-side / Inline) を 2 view まで満たす。3 view 目 (Field table) は本 page では適合性が低いため除外。
