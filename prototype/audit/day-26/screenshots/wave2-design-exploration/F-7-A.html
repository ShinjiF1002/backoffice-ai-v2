設計意図

SLA を文字で読める。 target は「目標 4 h」、elapsed は「経過 2:18 (57%)」のように 数値 + パーセンテージ を併記し、bar に頼らず 1 行で残時間が分かる。 色 token は既存 (slate / amber-soft / red-soft) のみ、進捗段階 ≥50%/100% で chip 色だけ切替。 approver 名は step hover の tooltip に格納し、stepper の visual rhythm を壊さない。 Delegate は Inbox 内で icon + tooltip の最小表現、誰から誰へ + 不在期間まで 1 hover で読める。

採用適性

既存 LifecycleStepper の 5 step 並びを変えず、stepper 下に縦方向で SLA / approver 行を増やすため 非破壊。Card 8 の "SLA per step visible + all approvers visible + delegate model" を tooltip + chip という低密度手段で全部満たす。 ただし step 数 ≥ 6 になると横スクロールが出るリスクあり (現 5 step は安全)。

// data/types.ts (拡張)
interface CaseLifecycleStepSpec {
  step: CaseLifecycleStep
  state: 'done' | 'current' | 'pending'
  slaTargetLabel: string                // `4 h` `24 h` `即時`
  slaTargetMinutes: number | 'instant'
  elapsedLabel?: string                 // `2:18` (current のみ)
  elapsedPercent?: number               // 0-200+ %
  approver?: { name: string; role: string }
}

// components/case/LifecycleStepper.tsx (拡張)
interface LifecycleStepperProps {
  current: CaseLifecycleStep
  specs?: CaseLifecycleStepSpec[]       // F-7 拡張、未指定なら従来通り step name のみ

// data/types.ts (CaseRecord 拡張)
