設計意図

色 (band) + 形 (icon) の dual cue で actor を区別。色覚 a11y を守りつつ、 16-22px の slim 高さで density-first の queue / breakdown / audit row に統合できる。 4px の ::before band は token --color-primary / slate / slate-subtle を再利用するため、Operational Premium Light の chip taxonomy (StatusBadge / FilterChip / MetaChip 3 系統) と一切競合しない。 sm/md 2 size は Inbox (sm) と AuditTrail row (md) で出し分け。

採用適性

既存 mock-audit の actor enum ('AI' | '入力者' | '承認者' | 'system') を 'agent' | 'human' | 'system' に薄く normalize するだけで使い回せる。一画面密度が高い Inbox queue で最も効く。 Card 1 の "icon prefix + color band で agent/human 区別" を装飾なしで字義通り実現。

Props shape (PR 2 Commit 5 reference)

// components/shared/ActorBand.tsx
type Actor = 'agent' | 'human' | 'system'

interface ActorBandProps {
  actor: Actor
  label?: string                // e.g. '田中 美咲' / 'AI 抽出 v2.3'
  size?: 'sm' | 'md'      // sm = h-5 (queue), md = h-7 (audit row)
  className?: string
}

// Color mapping (binding to existing tokens, no new tokens)
// agent  → --color-primary
// human  → slate-600
// system → slate-400
