設計意図

既存 AddressDiffBlock の inline diff は CaseReview の操作リズム (=各 field を縦に流し読み) と最も整合するため default として温存。 toggle で起動する Field table view は 5 field × 変更前 / 変更後 / 差分タイプ の matrix で、複数項目の同時 audit が要る監督者用途を満たす。 MetadataStrip は footer 直上の slate-50 anchor に置き、scroll-into-view (および 1 度の click ack) を gate にして承認 button を aria-disabled → active 化することで、Card 2 ★ の UX gate を装飾なしで成立させる。

採用適性

CaseReview の質感維持を最優先。AddressDiffBlock rewrite を避け、 toggle 一本だけで Field table を露出するため 既存 5 primitive (NextActionStrip / Disclosure 等) と干渉しない。安全側に倒した最小変更。

Props shape (PR 2 Commit 3 reference)

// components/case/DiffPreviewBlock.tsx
type DiffView = 'inline' | 'sideBySide' | 'fieldTable'

interface DiffPreviewBlockProps {
  fields: CaseField[]            // 5 field
  defaultView?: DiffView         // = 'inline' (案 A)
  availableViews?: DiffView[]    // = ['inline', 'fieldTable']
  onViewChange?: (v: DiffView) => void
}

// components/case/MetadataStrip.tsx
interface MetadataStripProps {
  changeAuthor?: string
  changeReason?: string
  confidence?: number
  affectedScope?: string
  reversibility?: 'Revertible' | 'Partial' | 'Irreversible'
  onAck?: () => void             // gate trigger for approval
}

Gate behaviour · 承認 button は initial aria-disabled="true"。MetadataStrip が viewport に scroll-in した瞬間 (IntersectionObserver) 、または 1 度 click された瞬間に active 化。下の 承認 button を試してみてください — メタデータ未確認では tooltip + shake、Strip 上に click すると active になります。
