設計意図

面積で識別。circle avatar の塗り + halo の二重円が、 row scroll 中 200ms 以下で actor type を脳が拾えるよう設計。 human だけ initial (2 文字) を text fallback として置き、誰の操作かを 1 row 内で完結する。 avatar のサイズは StatusBadge や FilterChip よりわずかに高く (24px)、 row の vertical rhythm を壊さない最大半径に揃えた。

採用適性

色覚 a11y は icon / initial が冗長 cue として効くが、row height が +4px 高くなる trade-off がある (32px → 36px)。 Inbox queue で「人の顔感」を出したい場合は強い差別化となる一方、density 重視 (Operational Premium Light の本流) との張力がある。 2 actor type の場合 (e.g. system event のみ) は overkill になりやすい。

Props shape (PR 2 Commit 5 reference)

// components/shared/ActorBand.tsx (案 B)
type Actor = 'agent' | 'human' | 'system'

interface ActorBandProps {
  actor: Actor
  label?: string                  // 表示文字列
  /** human のとき initial 抽出元、未指定なら label の先頭 2 字 */
  initial?: string
  size?: 'sm' | 'md'        // sm = 20px avatar, md = 24px (default)
  hideLabel?: boolean             // avatar のみ表示 (audit row 等)
  className?: string
}

// Color mapping
// agent  → primary fill + Bot white + indigo halo /12
// human  → slate-600 fill + initial white + slate halo /10
// system → slate-100 fill + Cog slate-500 + slate halo /12
