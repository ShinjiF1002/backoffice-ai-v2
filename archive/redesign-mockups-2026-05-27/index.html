Redesign Mockups — Backoffice AI v2 prototype

既存 prototype 9 画面 × 20 改善案 (HTML mockup)。3 問題対処の方向性を視覚で比較し、採用案を choose する reference set。

使い方

各画面の variant カードをクリックして mockup を新タブで開く。Cmd+クリックで複数 variant を並べて比較すると効率的です。

  1. 採用したい variant を画面ごとに 1 つ選ぶ (採用しない、保留も可)
  2. 採用後の Phase 2 で React 実装 → 既存 prototype 置き換え
  3. 本ディレクトリは reference として残し、Phase 2 後にも参照可能

3 問題 と対処方針

① 文字が多すぎる — instructional copy / 「次の実装段階で対応」 footer / 重複 chip を削減、accordion / 折りたたみ / drawer に逃がす

② 一目で次にやることが分からない — primary CTA の唯一性 + 大型化 + 色階層強化、case ID よりも顧客名 + action verb を hero に

③ 画面レイアウト — 等価重み 3 column → 1 hero + accordion へ、status group header で scan を支援、kanban / wizard で動線を visual に
